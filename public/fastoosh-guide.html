Everything you need to deploy a fresh instance, configure all third-party integrations, and manage every corner of the admin panel — from projects and tools to AI translations and payment analytics.

Setup Wizard

Navigate to /setup on a fresh deployment. The wizard runs once — it automatically locks itself and redirects to /admin/login as soon as an admin account exists. Run it immediately after cloning the project to a new Supabase project.

⚠️

One-time only. Do not run the Setup wizard on a live project that already has data. Doing so will attempt to overwrite the admin account and potentially seed duplicate sample content.

How to add secrets

Open your Supabase project dashboard

Go to supabase.com/dashboard and select your project.

Navigate to Edge Functions → Secrets

In the left sidebar: Edge Functions → Manage secrets. This is separate from the Database Vault.

Add each secret by name

Click Add new secret. Use the exact key names listed in the table below. The three Supabase secrets (SUPABASE_URL, SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY) are auto-injected and will already be ✓.

Click Refresh in the wizard

Return to the wizard and press the Refresh button — the checklist will re-ping the server and update statuses live.

③ OAuth Providers

Configure Google and/or Discord social sign-in. Both are optional — you can skip this step entirely and enable them later from Supabase → Authentication → Providers.

Callback URLs

Both providers require two URLs. The wizard shows them as copyable boxes:

Callback URLs

Supabase callback URL — paste into both provider portals as Redirect / Authorised URI:

https://<your-project-id>.supabase.co/auth/v1/callback

App callback URL — add to Supabase → Auth → URL Configuration → Redirect URLs:

https://<your-app-domain>/auth/callback

Google OAuth — step by step

Open Google Cloud Console

Go to console.cloud.google.com → APIs & Services → Credentials.

Create an OAuth 2.0 Client ID

Click Create Credentials → OAuth 2.0 Client ID. Choose Web application.

Add the Supabase callback URL

Under Authorised redirect URIs, paste the Supabase callback URL from the wizard.

Copy Client ID and Secret

After saving, Google shows a Client ID and Client Secret. Copy both.

Enable in Supabase

Go to Supabase → Auth → Providers → Google. Toggle Enable, paste the Client ID and Client Secret, then save.

Discord OAuth — step by step

Create a Discord Application

Go to discord.com/developers/applications and click New Application.

Add OAuth2 Redirect URI

In your app: OAuth2 → Redirects. Add the Supabase callback URL. Set scopes: identify and email.

Copy Client ID and Secret

From the OAuth2 → General page, copy the Client ID and Client Secret (click Reset Secret if needed).

Enable in Supabase

Go to Supabase → Auth → Providers → Discord. Toggle Enable, paste the credentials, then save.

ℹ️

The wizard uses the PKCE flow — no client secret is stored in the frontend. The flow works entirely through Supabase's auth service which handles token exchange server-side.

Security rules

  • Password must be at least 8 characters.
  • Both password fields must match exactly.
  • The account is created via supabase.auth.admin.createUser with email_confirm: true, so no email verification is required.
  • The admin session uses a UUID token stored in localStorage under admin_token, validated on every request via the X-Admin-Token header.
🔒

The SUPABASE_SERVICE_ROLE_KEY is never sent to the browser. Admin creation runs entirely through the Hono server which uses the service role key internally.

Style Tab

Live theme editor. Changes are applied instantly to the public site via CSS custom properties injected into :root. Saved to KV and persisted across sessions.

Mode

Toggle between Dark mode (default — deep black background) and Light mode. Switching modes loads preset safe defaults for that palette.

Color palette

  • Primary color — used for neon glows, active buttons, and accent gradients.
  • Secondary color — used for secondary accents and hover states.
  • Background color — the root background.
  • Card background — glassmorphism card base color.
  • Text color — primary text.
  • Muted text color — secondary / caption text.

The Color Palette Generator (ColorPaletteGenerator.tsx) provides preset harmony palettes (analogous, complementary, triadic) you can pick from. A live contrast ratio checker warns if text-on-background contrast falls below WCAG AA.

Typography

  • Heading font — 40+ Google Fonts curated for motion design studios (Orbitron, Space Grotesk, Bebas Neue, Syne, Playfair Display, etc.).
  • Body font — separate selection for paragraph text.

Fonts are dynamically loaded via a <link> tag injection — no build step required.

AI Translate button

Each content type + language combination has a ✨ Translate All button. It calls Gemini via the server endpoint /translate, which:

  1. Extracts all translatable fields (skipping IDs, URLs, slugs, dates).
  2. Sends them in one batch to Gemini with instructions to preserve tone and formatting.
  3. Returns the translated strings and populates all fields in the table.
  4. You review and can edit individual cells before saving.

Arabic RTL

When Arabic is selected, the entire public site layout switches to dir="rtl". All flex layouts, padding, and text alignment are mirror-flipped. The admin panel is unaffected.

Saving translations

Clicking Save writes the translation object to the KV store under the key translations:{lang}:{contentType}. The public pages check this key on every page load via fetchTranslations() and apply the override — no build step required.

💡

Translations are cached client-side for the session. If you update a translation and want to see it live immediately on the public site, open the page in an incognito window or do a hard refresh.

⚠️ Reset Tab — Danger Zone

A selective data-initialization panel designed to be used before going live. It lets you permanently wipe any combination of test-data categories while keeping every piece of real content — Projects, Tools, Team members, Client Logos, Home content, Style, SEO, and Translations — completely untouched.

🚨

All deletions are immediate and irreversible. There is no undo, no trash, no backup created by this panel. Run it on test data only, and never on a live production instance with real users unless you are 100% certain.

Server-side implementation

The Reset tab uses two dedicated endpoints on the Hono server (/supabase/functions/server/index.tsx):

EndpointMethodPurpose
/admin/reset/stats GET Returns live record counts for all 6 categories. Used to populate the card badges and the total-records warning in the confirmation modal.
/admin/reset POST Accepts { email, password, categories }. Re-verifies credentials, then deletes the selected categories in parallel. Returns a per-category report.

Both endpoints are protected by the requireAuth middleware (X-Admin-Token header check) in addition to the password re-verification in the POST handler.

Video Analytics category — detail

Video analytics are stored as individual KV entries, one per project video and one for the showreel:

  • project_video_stats:{projectId} — stores { views, totalWatchSeconds, avgWatchSeconds, lastViewed } for each project.
  • showreel_video_stats — same shape, for the Home page showreel embed.

Resetting this category deletes all project_video_stats:* entries plus the showreel_video_stats key. The Video Analytics sub-tab in Dashboard will immediately show zeroed counts after reset.

💡

To clear video data for a single project without touching everything else, you can use the standalone DELETE /admin/video-stats endpoint directly — but in normal usage, the Reset tab is the recommended approach since it gives you full visibility and a password confirmation gate.

Authentication Flow

Admin authentication

The admin panel uses a custom token system — separate from Supabase Auth. The flow:

Admin submits email + password at /admin/login

Server validates credentials against Supabase Auth

Uses supabase.auth.signInWithPassword(). If valid, generates a UUID session token and stores it in KV under admin_session:{token} with a TTL.

Token returned to browser and saved in localStorage

Key: admin_token.

Every admin API request sends the token

Via the custom header X-Admin-Token: <uuid>. The server validates it on each request.

Sign out invalidates the server-side session

POSTs to /logout, which deletes the KV entry. Token is also removed from localStorage.

Public user authentication

Visitors can sign up / log in on the Account page (/account). Supported methods:

  • Email + Password — with multi-layer security on signup (rate limiting, domain blocklist, disposable email check).
  • Google OAuth — PKCE flow via supabase.auth.signInWithOAuth({ provider: 'google' }).
  • Discord OAuth — PKCE flow via supabase.auth.signInWithOAuth({ provider: 'discord' }).
  • Forgot Password — sends a password-reset link via Resend. The reset URL lands on /reset-password.
ℹ️

OAuth callback handling is in /src/app/pages/AuthCallback.tsx. After the OAuth redirect, it exchanges the code for a session token and redirects the user to /account.

AI Features

AI Improve (inline)

Available on virtually every text field in the admin panel. The AIImproveModal accepts:

  • The current field value.
  • A context object — e.g. for a project description, context includes the project title, category, year, and client.
  • A field type hint — title, description, bio, headline, cta, etc.

It sends all this to the server endpoint /ai/improve, which calls Gemini with a tailored prompt and returns an improved version. The user can Accept, Regenerate, or Discard.

AI Translate (bulk)

In the Translations tab, the Translate All button sends a batch of up to 50 field key-value pairs to /translate. The server calls Gemini with a structured prompt that instructs it to return a JSON object with the same keys but translated values, preserving formatting, HTML entities, and tone.

AI SEO Generation

In the SEO tab, each page entry has a Generate All with AI action that auto-populates all 10 SEO fields (title, description, keywords, OG, Twitter, canonical, structured data) in one request.

Localization (i18n)

Setup

The site uses i18next with react-i18next. Locale files live in /src/locales/:

  • en.json — English (default, compiled into the app bundle).
  • fr.json — French.
  • ar.json — Arabic.

Runtime translations

Dynamic content (projects, tools, team bios, home copy) is translated at runtime via the KV store — the Translation tab writes to translations:fr:projects, etc. The public pages call fetchTranslations(lang, type) on mount and deep-merge the result over the English data.

Language switcher

A language picker in the site navigation lets visitors switch between EN / FR / AR. Switching triggers a re-fetch of translated content and flips the dir attribute to rtl for Arabic.

RTL support

All public pages use Tailwind's rtl: variant for mirrored layouts. Navigation icons, padding, flex directions, and text alignment all flip correctly. The admin panel is exempt.

Payments (Lemon Squeezy)

How checkout works

Visitor clicks "Buy" on a paid tool tier

The frontend builds a Lemon Squeezy checkout URL using the lemonSqueezyVariantId from the tool version and appends a custom checkout[custom][user_email] parameter if the user is logged in.

Lemon Squeezy handles payment

The user completes payment on the LS-hosted checkout page. No card data ever touches the Fastoosh server.

Webhook fires to the server

LS sends a order_created or subscription_created event to /webhooks/lemonsqueezy. The server verifies the signature using LEMON_SQUEEZY_WEBHOOK_SECRET, then records the order in the KV store and marks the user's license as active.

User gets access + email notification

The user's Account page shows their purchase under My Downloads. An admin notification email is sent via Resend.

Revenue Panel

The Dashboard → Revenue panel hits the LS REST API (/v1/orders) to fetch real-time revenue data: total, last 30 days, last 7 days, refunds, and individual order cards with status badges.

⚠️

Lemon Squeezy Variant IDs must be entered manually per tool version in the admin Tools form. You find them in your Lemon Squeezy dashboard under Products → Variants.
